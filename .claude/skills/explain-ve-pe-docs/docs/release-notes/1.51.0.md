## List of changes
### **New "Photo" tab on camera screen**
A new **Photo** tab appears at the bottom of the camera screen as the third tab. It allows users to capture a photo and directly launch the **Photo Editor SDK**, simplifying the photo-taking flow. 
> ❗ 
This tab is available **only when both the Photo Editor SDK and Video Editor SDK are integrated**
> ❗ 
The previous "Photo" tab, which captured a photo and converted it to a video, is now renamed to **Story**.

![](attachment:7e5ea969-3334-4a55-903a-324107331aa9:VE_PE_Cam_GIF_1_296x640_.gif)

### **Auto-Start Captions Processing**
Captions processing now begins automatically when the user opens the video editor screen. The SDK extracts audio from the video, transcribes it to text, and adds the text as text effects overlaying the video. This eliminates the need for users to manually navigate to the **Text** menu and select **Add Captions**, streamlining the caption creation workflow.

![](attachment:270a5013-0420-4b81-84de-bfcdc8245fad:VE_AutoCaptions_GIF_1_296x640_.gif)
### **Alignment Guidelines for Text and Sticker Effects**
Vertical and horizontal guidelines now appear when adding text or sticker effects to a video. These visual guides indicate when an effect is positioned close to the left, top, right, or bottom edge of the video frame, ensuring the effect remains fully within viewable bounds when the video is played.
![](attachment:f6ed23fd-8387-4f04-8353-4e5a5a246712:VE_Guides_GIF_1_296x640_.gif)

### Add Photo tab on camera screen
> ❗ 
**Requires Photo Editor SDK.** Contact Banuba if your license does not include it.
Add In your Koin module if not added previously to integrate Photo Editor  to Video Editor.
```Kotlin
 factory<OnImageEditorCallback> {
	 PhotoEditorHandler
 }
```
Use `preview` property of`ExportResult.Success` of your export video callback to obtain saved image result from Photo Editor.
### Configure **Auto-Start Captions Processing**
We added 2 new properties to `CaptionsConfig`
- `autoStartLanguageCode`  - certain language code to enable auto start captions i.e. “`en-US`”. Default value `null`
- `preferBottomPosition`  - whether to place captions at the bottom of the video. Default value `false`
### Changes in CameraConfig
New property `recordingModes` to `CameraConfig` is added in favor of removed  `CameraRecordingModesProvider` . 
You can adjust recording modes 
```Kotlin
CameraConfig(
	...
	recordingModes = listOf(RecordMode.Video, RecordMode.Story)
	
)
```
Default value `listOf(RecordMode.Video, RecordMode.Story)` .
> ❗ 
`Photo` tab is added automatically if Photo Editor is integrated.

`CustomIntegrationAppTheme` is set in `AndroidManifest.xml`
```XML
   <activity
            android:name="com.banuba.sdk.ve.flow.VideoCreationActivity"
            android:screenOrientation="portrait"
           ** android:theme="@style/CustomIntegrationAppTheme" **
            android:windowSoftInputMode="adjustResize"
            tools:replace="android:theme" />
```
**Now you can remove these styles and specify **`**VideoCreationTheme**`** directly in **`**AndroidManifest.xml**`**.**
```XML
   <activity
            android:name="com.banuba.sdk.ve.flow.VideoCreationActivity"
            android:screenOrientation="portrait"
           ** android:theme="@style/**VideoCreationTheme**"**
            android:windowSoftInputMode="adjustResize"
            tools:replace="android:theme" />
```
