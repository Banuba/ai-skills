## List of changes
### 1-second timeline increments

Now the video editing timeline is split into 1-second increments and displays milliseconds as well. This change will give users more precision when modifying the content and set the duration of various effects. The new timeline appearance applies to Banuba Video Editor SDK UI V2 only.
![](attachment:07f80da8-6dbe-4c80-97a5-7544a758af3a:VE_milisec_296x640_.gif)

### Deeper Photo Editor integration
Users can now switch from Video Editor SDK to Photo Editor SDK and edit a picture. Then they can go back, and turn this picture into a video. Creating interesting content has never been easier even for users who know nothing about editing.
> ❗ 
The feature is disabled by default

> ❗ Please check out [Pull Request ](https://github.com/Banuba/ve-sdk-android-integration-sample/pull/319/files) with migration changes.
```Groovy

implementation "com.banuba.sdk:ffmpeg:5.3.0"
implementation "com.banuba.sdk:camera-sdk:${banubaSdkVersion}"

implementation "com.banuba.sdk:core-sdk:${banubaSdkVersion}"
implementation "com.banuba.sdk:core-ui-sdk:${banubaSdkVersion}"
```

### Enable Deeper Photo Editor integration feature
Add to your Video Editor Koin module.
> ❗ 
Your license must include FACE AR and PHOTO EDITOR
```Kotlin
 private class SampleIntegrationKoinModule {
    val module = module {
	    ...
	     
        single {
            EditorConfig(
	            ...
	            supportPhotoEditing = true
            )
        }

        factory<OnImageEditorCallback> {
            PhotoEditorHandler
        }
    }
 }
 
```
